Why updates were necessary

I thought header looked quite busy with background color and not so much breathing space. I felt, I should try to improve somehow. Latest updates

  • Contact page
  • Comments on html and js files
  • Transition properties

Why updates were necessary

As a user, you must be able to contact site owner for possible feedback or queries. Which I didn't think about when designing site. So contact page was necessary. After going back to redesign, I felt that it was hard to read codes in html and js files, particulary was hard to find out for what or where they belong to. So description or commenting was necessary for readability. Some other issues I came up was, button on hover snapping back which kinda didn't feel smooth user experience.

How changes were Implemented

As per teacher's feedback, contact page was missing for site. I added contact page and I didn't find place for contact button in navigation, since it wasn't actually part of design. So I added button in footer section which takes user to contact form. I further went on to clean up some messy code and added comments on each section, which helps for readability specially when there is lots of content in html pages.

Some transition properties added, so button did notsnap back when user leaves button, rather add smooth transition which I think feels much better as user.
